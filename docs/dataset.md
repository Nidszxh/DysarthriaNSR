## TORGO Corpus

### Overview

TORGO is a **dysarthric speech corpus** developed at the University of Toronto by Rudzicz, Namasivayam, and Bhasha (2012). It contains speech from individuals with cerebral palsy and amyotrophic lateral sclerosis (ALS/MND), paired with matching control speakers.

| Property | Value |
|----------|-------|
| Published by | University of Toronto (2012) |
| Access | Available via HuggingFace Hub (`abnerh/TORGO-database`) and academic request |
| License | Academic research use |
| Languages | English |
| Speech type | Read speech (prompted sentences) |

### Severity Mapping

Continuous severity scores in `TORGO_SEVERITY_MAP` (from `src/utils/config.py`), derived from Rudzicz et al. (2012) percent-word-correct intelligibility ratings:
`severity = (1 - intelligibility/100) * 5.0`

Values span `[0.0, 5.0]` where `0.0 = 100% intelligible` and `5.0 = 0% intelligible`.

## Downloading the Dataset

### Method 1 — `download.py` (Recommended)

```bash
python src/data/download.py
```

Uses HuggingFace `datasets` to fetch audio metadata from `abnerh/TORGO-database`, then downloads the WAV files. Saves files to `data/raw/audio/` with naming convention:

```
data/raw/audio/{speaker_id}_{md5_hash_of_hf_path}_{original_filename}.wav
```

The MD5 hash of the HuggingFace path is used as a stable identifier to match dataset entries to local files (critical for manifest generation later).

### Method 2 — Manual Download

If access to the HuggingFace repository is unavailable, the TORGO corpus can be obtained from the [TORGO dataset page](http://www.cs.toronto.edu/~complingwiki/index.php/TORGO) (academic request required).

After downloading manually, files must be renamed to match the expected convention. Contact the data curator for details.

## Manifest Structure

The manifest CSV at `data/processed/torgo_neuro_symbolic_manifest.csv` is the central data table. It is generated by `python src/data/manifest.py` and contains one row per utterance.

### Column Descriptions

| Column | Type | Description |
|--------|------|-------------|
| `sample_id` | str | Unique identifier: `{speaker}_{session}_{mic}_{n}` |
| `path` | str | Absolute path to `.wav` file |
| `speaker` | str | Speaker ID (e.g., `M01`, `FC03`) |
| `status` | int | 1=dysarthric, 0=control |
| `label` | int | Alias for `status` |
| `transcript` | str | Raw text prompt |
| `phonemes` | str | Space-separated stress-agnostic ARPABET sequence |
| `manner_classes` | str | Space-separated manner-of-articulation labels per phoneme |
| `place_classes` | str | Space-separated place-of-articulation labels per phoneme |
| `voice_classes` | str | Space-separated voicing labels per phoneme |
| `phn_count` | int | Number of phonemes in sequence |
| `phonemes_per_sec` | float | Phoneme rate (phn_count / duration) |
| `duration` | float | Audio duration in seconds |
| `rms_energy` | float | Mean RMS energy (from librosa) |
| `gender` | str | `F` (female) or `M` (male), inferred from speaker ID |

**Example row:**
```
sample_id: M01_s1_mic1_001
path: /home/user/DysarthriaNSR/data/raw/audio/M01_abc123_s1_mic1_001.wav
speaker: M01
status: 1
transcript: "Put the book on the table"
phonemes: P UH T DH AH B UH K AH N DH AH T EY B AH L
manner_classes: stop vowel stop fricative vowel stop vowel stop vowel nasal fricative vowel stop diphthong stop vowel liquid
place_classes: bilabial central alveolar dental central bilabial central velar central alveolar dental central alveolar front bilabial central alveolar
voice_classes: voiceless vowel voiceless voiced vowel voiceless vowel voiceless vowel voiced voiced vowel voiceless voiced voiceless vowel voiced
phn_count: 17
phonemes_per_sec: 4.2
duration: 4.05
rms_energy: 0.034
gender: M
```

### Phoneme Vocabulary

**44 stress-agnostic ARPABET phonemes** (stress markers 0/1/2 stripped by `normalize_phoneme()`):

| Category | Phonemes |
|----------|----------|
| Stops (voiceless) | P, T, K, CH, B |
| Stops (voiced) | B, D, G, JH |
| Fricatives | F, TH, S, SH, HH, V, DH, Z, ZH |
| Nasals | M, N, NG |
| Liquids | L, R |
| Glides | W, Y |
| Vowels | IY, IH, EH, AE, AH, AO, OW, UH, UW |
| Diphthongs | EY, AY, OY, AW |
| Syllabic | ER, AX |
| Reduced | AH, IX |

Plus 3 special tokens: `<BLANK>` (ID=0), `<PAD>` (ID=1), `<UNK>` (ID=2).

**Total vocabulary size:** 47 tokens.

The vocabulary is built at dataset initialization time from the sorted unique phonemes in the manifest. `normalize_phoneme()` (in `config.py`) strips stress digits from all tokens at manifest-build, dataset-init, and decode time. Vocabulary must be consistent across manifest generation and training runs (detected by `on_load_checkpoint`).

### Articulatory Features

**Manner of articulation classes** (used by manner_head):
```
stop, fricative, affricate, nasal, liquid, glide, vowel, diphthong
```

**Place of articulation classes** (used by place_head):
```
bilabial, labiodental, dental, alveolar, postalveolar, palatal, velar,
glottal, labio-velar, front, back, central
```

**Voicing classes** (used by voice_head):
```
voiced, voiceless, vowel
```

These are defined in `PHONEME_DETAILS` in `src/data/manifest.py` (used for G2P annotation) and in `PHONEME_FEATURES` in `src/models/model.py` (used to build the static constraint matrix `C_static`). Both dictionaries must remain in sync (B23 fix: labels were aligned to IPA standard; key fix was `SH/ZH/CH/JH: palatal → postalveolar`, `R: palatal → alveolar`, `W: bilabial → labio-velar`).

## Data Filtering

The manifest generation applies quality filters to exclude uninformative or corrupt entries:

| Filter | Threshold | Reason |
|--------|-----------|--------|
| Minimum phoneme count | ≥ 2 | Single-phoneme utterances unreliable for CTC |
| Maximum phoneme rate | ≤ 20 phonemes/sec | Transcription errors / corrupted metadata |
| Minimum duration | > 0.1s | Sub-threshold frames produce degenerate CTC inputs |
| Non-empty transcript | Required | G2P requires non-empty text |
| Non-empty G2P output | Required | `g2p_en` must produce ≥ 1 phoneme |

After filtering, the manifest contains approximately 16,531 utterances.

## Data Splits

**Speaker-stratified split** (guarantees no speaker appears in multiple splits):

| Split | Fraction | Approx. speakers |
|-------|----------|-----------------|
| Train | 70% | ~10 speakers |
| Validation | 15% | ~2 speakers |
| Test | 15% | ~2–3 speakers |

Split is seeded via `np.random.seed(config.experiment.seed)` before speaker shuffle, ensuring consistent assignment across runs with the same seed.

**For LOSO-CV**, the split is performed per fold: the test set is exactly 1 held-out speaker; the validation set is 2 speakers drawn from the remaining 14. This ensures no speaker's data is seen during training of the fold that holds them out.

## Known Dataset Issues

| Issue | Detail |
|-------|--------|
| Pilot / duplicate utterances | TORGO includes repeated prompts (session 1 and session 2 use similar/identical sentences). No de-duplication is applied; utterances are distinct even if transcripts match. |
| Speaker FC02 in manifest | FC02 appears in the TORGO corpus but may have reduced recordings. Verify `speaker == 'FC02'` rows in the manifest. |
| Inverted dysarthric/control PER | Single-split test sets of 2 speakers produce unreliable stratified statistics. This is a sampling artefact, not a data issue. |
| Control severity = 0.0 | All control speakers receive `severity=0.0` (100% intelligibility). This means all controls form an identical severity bucket in ordinal contrastive loss (partially mitigated by continuous TORGO_SEVERITY_MAP for dysarthric speakers). |
